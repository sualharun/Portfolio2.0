# Vercel Deployment Guide

## Frontend: Deploy to Vercel

### Step 1: Push to GitHub
```bash
git add .
git commit -m "Configure Vercel deployment"
git push origin main
```

### Step 2: Connect to Vercel
1. Go to [vercel.com](https://vercel.com)
2. Sign in with GitHub
3. Click **Add New...** → **Project**
4. Select your `Portfolio2.0` repository
5. **Framework:** Vite (auto-detected)
6. **Settings → Environment Variables**, add:
   - **Name:** `VITE_API_URL`
   - **Value:** `https://your-backend-url.com` (see Step 3)

### Step 3: Deploy Backend

Choose **ONE** of these options:

#### Option A: Render (Recommended — Free Tier)
1. Go to [render.com](https://render.com)
2. Click **New +** → **Web Service**
3. Connect your GitHub repo
4. **Settings:**
   - **Name:** `portfolio-api`
   - **Environment:** Java
   - **Build Command:** `mvn clean package`
   - **Start Command:** `java -jar target/portfolio-1.0.0.jar`
5. **Environment Variables:**
   - `PORT=10000` (Render assigns dynamic ports)
6. Deploy
7. Copy your Render URL (e.g., `https://portfolio-api.onrender.com`)

#### Option B: Railway (Pay-as-you-go, $5/month free credits)
1. Go to [railway.app](https://railway.app)
2. Click **New Project** → **Deploy from GitHub**
3. Select your repo
4. **Service:** Java 21
5. **Build Command:** `mvn clean package`
6. Add env var `PORT=8080`
7. Deploy
8. Copy your Railway URL

#### Option C: Heroku (Paid — ~$5–7/month)
1. Install Heroku CLI
2. Run:
   ```bash
   heroku login
   heroku create portfolio-api
   git push heroku main
   ```
3. Add `Procfile` to your repo root:
   ```
   web: java -Dserver.port=$PORT $JAVA_OPTS -jar target/portfolio-1.0.0.jar
   ```

### Step 4: Update Vercel Environment Variable
In Vercel project settings, update `VITE_API_URL` to your backend URL.

Example: `https://portfolio-api.onrender.com`

### Step 5: Redeploy Frontend
Go to Vercel → Your Project → **Deployments** → **Redeploy** latest or push a new commit.

## Production Domain (Optional)

### Custom Domain on Vercel
1. Vercel Dashboard → Project → **Settings** → **Domains**
2. Add your custom domain (e.g., `sualharun.com`)
3. Update DNS at your registrar to point to Vercel

### Update CORS for Custom Domain
In `src/main/java/com/sualharun/portfolio/config/WebConfig.java`, the `https://sualharun.com` is already included.

## Testing Locally

**Terminal 1 — Backend:**
```bash
PORT=8080 mvn spring-boot:run
```

**Terminal 2 — Frontend (with backend URL):**
```bash
cd frontend
VITE_API_URL=http://localhost:8080 npm run dev
```

Or set it in `.env.local`:
```env
VITE_API_URL=http://localhost:8080
```

## Troubleshooting

- **CORS errors?** Check `WebConfig.java` includes your frontend domain.
- **404 on `/api/*`?** Verify `VITE_API_URL` is correctly set in Vercel.
- **H2 database lost?** Render/Railway restart clears the file-based H2. Consider migrating to PostgreSQL or MySQL for production.

## Next: Upgrade H2 to PostgreSQL (Optional)

For production, replace H2 with PostgreSQL:

1. Add dependency to `pom.xml`:
   ```xml
   <dependency>
       <groupId>org.postgresql</groupId>
       <artifactId>postgresql</artifactId>
       <scope>runtime</scope>
   </dependency>
   ```

2. Update `application.properties`:
   ```properties
   spring.datasource.url=jdbc:postgresql://${DB_HOST}:5432/${DB_NAME}
   spring.datasource.username=${DB_USER}
   spring.datasource.password=${DB_PASSWORD}
   spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
   ```

3. Add environment variables on Render/Railway.
